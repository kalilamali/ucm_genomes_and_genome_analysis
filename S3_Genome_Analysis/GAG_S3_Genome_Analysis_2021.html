0. Reference genome of SARS-CoV 2.

WIV04 is the official reference sequence employed by GISAID. WIV04 was chosen because of its high-quality genome sequence and because it represented the consensus of a handful of early submissions for the betacoronavirus responsible for COVID-19. (Pilailuk et al 2020) WIV04 is representative of and identical to the early outbreak sequences. WIV04 was isolated by the Wuhan Institute of Virology from a clinical sample of a bronchoalveolar lavage fluid (BALF) collected at the Wuhan Jinyintan Hospital in Hubei Province on 30th December 2019 from a symptomatic patient, a retailer working at the Huanan Seafood Wholesale Market. WIV04 sequence accession in ENA browser is MN996528.1. Visit ENA.

2. Quality check using FastQC

We will use FastQC to check the quality of our data, this is important to figure out what kind of limitations we might have to analyse our data and the selection of parameters. Poor quality data will dramatically impact on the interpretation of data and results.


# 1-2 minute/s
# prefix="ERR4780723"
prefix="HERE YOUR RUN ACCESSION"

fastqc ${prefix}*

3. Index reference genome of SARS-CoV2 (WIV04)

We will first map the data to the SARS-CoV2 reference genome using BWA. To speed up the mapping the reference genome MUST be indexed. BWA uses an FM-Index which is built around the Burrows-Wheeler transformation.

# < 1 minute
bwa index MN996528.1.fasta

It is also useful to build an index of the FASTA reference file using SAMtools to allow a quick extraction of subsequences from the reference genome.


# < 1 minute
samtools faidx MN996528.1.fasta

4. Mapping reads with BWA-MEM

We will use BWA-MEM for the alignment of the processed reads to the reference genome. BWA-MEM is a fast software package for mapping low-divergent sequences against a large reference genome, such as human. So once the index has been built we can map the paired-end FASTQ data against the reference and convert it to BAM.

# 1-2 minutes
bwa mem MN996528.1.fasta ${prefix}_1.fastq.gz ${prefix}_2.fastq.gz -o ${prefix}.sam

Transform and sort SAM files into BAM files

# 1 minute
samtools view -o ${prefix}.bam -Shb ${prefix}.sam

head ${prefix}.sam

Have a look into the generated bam file to check its contents. You can read it using samtools

# 1 minute
samtools view ${prefix}.bam | head

We use samtools to sort by coordinates our bam file, and we build a new bam file.

# 1 minute
samtools sort ${prefix}.bam -o ${prefix}.sorted.bam

samtools index ${prefix}.sorted.bam

5. Mark Duplicates and Alignment Quality Control

Unless you are using a PCR-free library, PCR duplicates are common in DNA-sequencing and should be flagged prior to variant calling.

# 2 minutes
picard MarkDuplicates INPUT=${prefix}.sorted.bam OUTPUT=${prefix}.dedup.sorted.bam METRICS_FILE=${prefix}_metrics.txt REMOVE_DUPLICATES=true

6. Variant Calling

Once the alignment is sorted and duplicates are marked we can run a variant caller such as FreeBayes to scan the alignments for differences compared to the reference.

# 1-2 minutes

bcftools mpileup -Ou -f MN996528.1.fasta ${prefix}.dedup.sorted.bam | bcftools call --ploidy 1 -Oz -m -v -o ${prefix}.vcf.gz

bcftools index ${prefix}.vcf.gz

7. Get the FASTA sequence of the isolate


cat MN996528.1.fasta | bcftools consensus ${prefix}.vcf.gz > ${prefix}.fasta

8. Explore the VCF file

gunzip ${prefix}.vcf.gz

# Check contents
head -n 100 ${prefix}.vcf

# Count the number of variants
wc -l ${prefix}.vcf

bcftools query -f '[%CHROM\t%POS\t%REF\t%ALT\t%SAMPLE\t%GT\n]' ${prefix}.vcf > ${prefix}_isolate_alleles.txt

9. Visualize your alignement with IGV

Integrative Genomics Viewer (IGV) is a high-performance, easy-to-use, interactive tool for the visual exploration of genomic data. Run IGV in the terminal, load the reference genome in FASTA format and load the BAM file of sorted coordinates ${prefix}.sorted.bam file.

# RUN IGV program
igv
